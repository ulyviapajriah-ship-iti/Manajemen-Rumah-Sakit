1. Analisis Kompleksitas Sistem (Materi: Big O Notation)

Modul Operasional Struktur Data / Algoritma Time Complexity (Worst Case) Keterangan Implementasi Sistem
Pendaftaran Antrean Klinik Queue (FIFO Array-based) O(1) Menambahkan data pasien baru ke ujung belakang barisan (Enqueue / Push).
Gudang Resep Apotek Stack (LIFO Array-based) O(1) Menaruh (Push) dan membatalkan (Pop) resep dari tumpukan paling atas.
Penyimpanan Rekam Medis Double Linked List O(n) Navigasi rekam medis dua arah lewat pointer PREV & NEXT dengan alokasi simulasi memori.
Prioritas Ranjang UGD Bubble Sort Algorithm O(n²) Mengurutkan keparahan pasien UGD secara sekuensial dengan membandingkan node bersebelahan.
Kalkulator Jaminan JKN Fungsi Rekursif O(n) Mengevaluasi diskon biaya bertingkat dengan memanggil dirinya sendiri di memori stack.

2. Perancangan Logika Program (Materi: Flowchart & Pseudocode)

PSEUDOCODE LOGIKA BUBBLE SORT PADA TRIASE UGD:

PROCEDURE bubble_sort_triage(ARRAY_PATIENTS pasien)
    DECLARE n = panjang_array(pasien)
    FOR i = 0 TO n - 1 DO
        FOR j = 0 TO n - i - 2 DO
            // Nilai prioritas kecil berarti kondisi makin darurat (kritis)
            IF pasien[j].priority > pasien[j+1].priority THEN
                // Swap/Tukar posisi elemen bersebelahan
                DECLARE temp = pasien[j]
                pasien[j] = pasien[j+1]
                pasien[j+1] = temp
            ENDIF
        ENDFOR
    ENDFOR
    PRINT "Pasien terurut berdasarkan tingkat urgensi"
ENDPROCEDURE
